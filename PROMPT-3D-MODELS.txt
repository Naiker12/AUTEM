========================================================================
  AUTEM — GUÍA PARA GENERAR MODELOS 3D (.glb / .usdz) PARA AR WEB
========================================================================

OBJETIVO:
Generar archivos .glb (Android/Desktop) y .usdz (iPhone) de cada propiedad
para que la experiencia de Realidad Aumentada funcione en el sitio web.

========================================================================
PROPIDADES QUE NECESITAN MODELO 3D (badge "AR Ready"):
========================================================================

1. The Horizon Suite    → the-horizon-suite.glb / .usdz
2. Villa Atlántico      → villa-atlantico.glb / .usdz
3. Penthouse Solemar    → penthouse-solemar.glb / .usdz
4. The Summit Penthouse → the-summit-penthouse.glb / .usdz

Empezar con UNA sola propiedad piloto (The Horizon Suite es la mejor candidata).

========================================================================
OPCIÓN A — GENERAR CON IA (gratis o de bajo costo)
========================================================================

Herramientas que convierten IMÁGENES a modelos 3D:

1. TripoSR (gratuito, open source)
   - Web: https://huggingface.co/spaces/stabilityai/TripoSR
   - Sube una foto del render de la propiedad
   - Descarga el resultado como .glb
   - Luego exporta a .usdz con un conversor online

2. Meshy.ai (freemium)
   - Web: https://meshy.ai
   - Sube imagen → genera modelo 3D
   - Exporta .glb directamente
   - Para .usdz: usar conversor online

3. CSM.ai (Common Sense Machines)
   - Web: https://3d.csm.ai
   - Similar a Meshy, genera desde imagen
   - Exporta .glb

4. Luma AI (Genie)
   - Web: https://lumalabs.ai/genie
   - Genera modelos desde texto o imagen
   - Exporta .glb

CONVERSOR .glb → .usdz:
- Web: https://product.dev/tools/glb-to-usdz/
- O: https://www.model-viewer.io/editor/
- Sube el .glb, descarga el .usdz

========================================================================
OPCIÓN B — RENDER 3D PROFESIONAL (recomendado para calidad)
========================================================================

Si ya tienes renders 3D de la propiedad (probablemente en Blender, 3ds Max,
o Cinema 4D), pide al diseñador 3D que exporte directamente:

- Formato .glb (GL Transmission Binary)
- Formato .usdz (Universal Scene Description para Apple)
- Escala real (1 unidad = 1 metro)
- Texturas embebidas en el archivo
- Triángulos: menos de 500K para que cargue rápido en web

PROMPT PARA ENVIAR AL DISEÑADOR 3D:
------------------------------------------------------------------------
"Necesito exportar el modelo 3D de [NOMBRE PROPIEDAD] para realidad
aumentada web. Por favor genera:

1. Un archivo .glb (GL Binary) con:
   - Escala real (1 unidad = 1 metro)
   - Texturas PBR embebidas (no externas)
   - Menos de 500,000 triángulos
   - Nombre del archivo: [slug-propiedad].glb

2. Un archivo .usdz (para Apple AR Quick Look) con:
   - Las mismas especificaciones
   - Nombre del archivo: [slug-propiedad].usdz

Estos archivos se usarán con <model-viewer> de Google para AR web.
La propiedad se verá en la cámara del celular del usuario, así que
la escala debe ser exacta (ej: una villa de 320m² debe verse como
una villa de 320m², no como un juguete)."
------------------------------------------------------------------------

========================================================================
OPCIÓN C — BLENDER (si sabes modelar)
========================================================================

1. Abre el modelo en Blender
2. File → Export → glTF 2.0 (.glb/.gltf)
   - Format: glTF Binary (.glb)
   - Include: Selected Objects (o All)
   - Transform: +X Forward, +Z Up
   - Materials: Export
   - Images: Automatic, Max 2048px
3. Para .usdz: usar el add-on "USDZ Tools" de Apple o conversor online

========================================================================
CHECKLIST DE VERIFICACIÓN (después de colocar los archivos):
========================================================================

[ ] El archivo .glb se abre en https://modelviewer.dev/editor/
[ ] El modelo se ve con escala correcta (no gigante ni miniatura)
[ ] Las texturas se ven bien (no blancas ni rotas)
[ ] Al abrir la página /ar/the-horizon-suite desde un iPhone:
    - Aparece el modelo 3D girando
    - Aparece un botón de AR
    - Al tocarlo, abre la cámara y coloca el modelo en el suelo
[ ] Al abrir desde Android:
    - Mismo comportamiento
    - El botón de AR usa Scene Viewer nativo
[ ] El modelo aparece con sombra en el suelo (contact shadow)
[ ] Se puede rotar con el mouse (desktop) o dedos (móvil)

========================================================================
NOTAS IMPORTANTES:
========================================================================

- Un solo modelo .glb típico pesa entre 1-10 MB (depende de texturas)
- Para AR web, menos de 5 MB es ideal (carga rápida en datos móviles)
- Si el modelo es muy pesado, reducir texturas a 1024x1024 px
- El .usdz es solo para iPhone/iPad — sin él, iOS no puede mostrar AR
- Los .glb y .usdz deben tener el MISMO modelo, solo cambia el formato
